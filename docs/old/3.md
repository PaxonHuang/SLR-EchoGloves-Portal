## 1. 设计 MQTT 数据格式和 Topic

### 1.1 Topic 设计（经典物联网分层）

假设你的设备ID为 `glove001`，产品名为 `signglove`，用户ID为 `user123`，推荐如下 topic 结构：

- **设备上报数据（设备→云/后端）**  
  ```
  signglove/user123/glove001/data
  ```
- **设备状态上报**  
  ```
  signglove/user123/glove001/status
  ```
- **云端/后端下发控制指令**  
  ```
  signglove/user123/glove001/cmd
  ```
- **云端/后端下发OTA升级**  
  ```
  signglove/user123/glove001/ota
  ```

> 你也可以用 `/device/{device_id}/data` 这种更通用的格式，核心是“产品/用户/设备/功能”分层。

---

### 1.2 数据格式（JSON，标准物联网风格）

**数据上报包示例：**
```json
{
  "timestamp": 1721212345,
  "device_id": "glove001",
  "user_id": "user123",
  "fingers": [12.3, 45.6, 78.9, 23.4, 56.7],  // 五指弯曲角度，单位度
  "imu": {
    "accel": [0.01, -0.02, 9.81],              // 三轴加速度
    "gyro": [0.1, 0.2, 0.3]                    // 三轴角速度
  },
  "battery": 87,                               // 电量百分比
  "signal": -65                                // WiFi/BLE信号强度
}
```
- **timestamp**：Unix时间戳，秒或毫秒
- **fingers**：五指弯曲角度数组
- **imu**：IMU原始数据
- **battery**：电池电量
- **signal**：信号强度

**状态包示例：**
```json
{
  "timestamp": 1721212345,
  "device_id": "glove001",
  "status": "online"
}
```

**指令包（下发）示例：**
```json
{
  "cmd": "vibrate",
  "params": { "duration": 500 }
}
```

## 2. ESP32-S3 端多任务并发架构设计

### 2.1 任务划分

- **传感器采集任务**（I2C/SPI/ADC，定时采集五指和IMU）
- **数据处理任务**（滤波、融合、打包JSON）
- **MQTT通信任务**（连接、上报、接收指令）
- **状态监控任务**（电量、信号、设备状态）
- **OTA/控制任务**（响应云端指令）

### 2.2 推荐开发环境

- **ESP-IDF（官方C/C++）** 或 **Arduino-ESP32（C++）**
- 推荐用 ESP-IDF，任务调度更强大

## 3. ESP32-S3 端代码示例（ESP-IDF风格，FreeRTOS多任务）

### 3.1 主要依赖
- `esp-mqtt`（官方MQTT库）
- `cJSON`（JSON打包）
- `driver/i2c.h`、`driver/adc.h`（传感器采集）

### 3.2 代码结构（伪代码+关键实现）

```c
// main.c
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "mqtt_client.h"
#include "cJSON.h"
#include "driver/i2c.h"
#include "driver/adc.h"
#include "esp_wifi.h"
#include "esp_log.h"
#include <string.h>
#include <time.h>

#define DEVICE_ID "glove001"
#define USER_ID   "user123"
#define MQTT_URI  "mqtt://broker.hivemq.com" // 替换为你的MQTT服务器

static esp_mqtt_client_handle_t mqtt_client = NULL;

// 采集数据结构
typedef struct {
    float fingers[5];
    float accel[3];
    float gyro[3];
    int battery;
    int signal;
} glove_data_t;

static glove_data_t glove_data;

// 采集任务
void sensor_task(void *pvParameters) {
    while (1) {
        // 采集五指弯曲角度
        for (int i = 0; i < 5; i++) {
            glove_data.fingers[i] = read_finger_angle(i); // 伪函数
        }
        // 采集IMU
        read_imu(glove_data.accel, glove_data.gyro); // 伪函数
        // 采集电量
        glove_data.battery = read_battery(); // 伪函数
        // 采集信号
        glove_data.signal = get_wifi_rssi(); // 伪函数
        vTaskDelay(pdMS_TO_TICKS(20)); // 50Hz
    }
}

// 数据上报任务
void mqtt_publish_task(void *pvParameters) {
    char topic[128];
    char payload[512];
    while (1) {
        snprintf(topic, sizeof(topic), "signglove/%s/%s/data", USER_ID, DEVICE_ID);
        cJSON *root = cJSON_CreateObject();
        cJSON_AddNumberToObject(root, "timestamp", time(NULL));
        cJSON_AddStringToObject(root, "device_id", DEVICE_ID);
        cJSON_AddStringToObject(root, "user_id", USER_ID);

        cJSON *fingers = cJSON_CreateFloatArray(glove_data.fingers, 5);
        cJSON_AddItemToObject(root, "fingers", fingers);

        cJSON *imu = cJSON_CreateObject();
        cJSON_AddItemToObject(imu, "accel", cJSON_CreateFloatArray(glove_data.accel, 3));
        cJSON_AddItemToObject(imu, "gyro", cJSON_CreateFloatArray(glove_data.gyro, 3));
        cJSON_AddItemToObject(root, "imu", imu);

        cJSON_AddNumberToObject(root, "battery", glove_data.battery);
        cJSON_AddNumberToObject(root, "signal", glove_data.signal);

        char *json_str = cJSON_PrintUnformatted(root);
        esp_mqtt_client_publish(mqtt_client, topic, json_str, 0, 1, 0);

        cJSON_Delete(root);
        free(json_str);

        vTaskDelay(pdMS_TO_TICKS(100)); // 10Hz上报
    }
}

// MQTT事件回调
static esp_err_t mqtt_event_handler_cb(esp_mqtt_event_handle_t event) {
    // 处理连接、断开、消息下发等
    return ESP_OK;
}

// MQTT连接任务
void mqtt_task(void *pvParameters) {
    esp_mqtt_client_config_t mqtt_cfg = {
        .uri = MQTT_URI,
    };
    mqtt_client = esp_mqtt_client_init(&mqtt_cfg);
    esp_mqtt_client_register_event(mqtt_client, ESP_EVENT_ANY_ID, mqtt_event_handler_cb, NULL);
    esp_mqtt_client_start(mqtt_client);
    vTaskDelete(NULL);
}

void app_main(void) {
    // 初始化WiFi、I2C、ADC等
    // ...
    xTaskCreate(sensor_task, "sensor_task", 4096, NULL, 5, NULL);
    xTaskCreate(mqtt_task, "mqtt_task", 4096, NULL, 6, NULL);
    xTaskCreate(mqtt_publish_task, "mqtt_publish_task", 4096, NULL, 4, NULL);
}
```
